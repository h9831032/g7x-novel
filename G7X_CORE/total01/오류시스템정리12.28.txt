## 1) 게이트는 있는데도 왜 이래?

게이트는 “문 앞 경비원”이야.
근데 경비원이 **어제는 A를 잡고 오늘은 A를 못 잡으면** 그 경비원은 실전에서 못 써.

그래서 우리가 한 건 이거야:

* **정답지(guard_cases)**: “이런 상황이면 PASS/WARN/SANDBOX/BLOCK 해야 한다”가 적힌 문제집
* **채점기(pytest/runner)**: 게이트가 문제집대로 판정하는지 매번 검사
* **줄자(meter_tick / receipt)**: “몇 번 돌렸고, 얼마나 걸렸고, 진짜로 호출했는지” 영수증

즉, **게이트 자체가 아니라 ‘게이트가 흔들리지 않게 고정하는 작업’**이라 고생이 난 거야.

## 2) 지금 말하는 ‘가드’가 줄자 가드냐?

둘 다 “가드”지만 역할이 달라.

* **게이트 가드(오류시스템/semantic_guard)**
  소설 문장을 보고 “이건 세계관 법 위반이냐?” 판정하는 애
  (PASS/WARN/SANDBOX/BLOCK)

* **줄자 가드(MeterGuard)**
  게이트가 뭘 했는지 “영수증이 제대로 남았냐?” 검사하는 애
  (ms/턴/포인터/receipt가 없으면 INVALID로 빼버림)

초딩버전으로:

* 게이트 가드 = **경찰**
* 줄자 가드 = **바디캠 + 사건번호 기록원**
  (경찰이 잡았다는 증거 없으면 “그 체포는 인정 안 함”)

### 1. 지금 한 게 뭐야? (PHASE 2 완공)

단순한 '단어 필터'가 아니라, 소설의 **맥락(Context)을 이해하는 지능형 검문소**를 만든 거야.

### 2. 왜 이전에도 했던 걸로 고생한 거야? (진짜 원인)

형아 말대로 이전 프로젝트 파일들에도 비슷한 형식이 있었지. 그런데 이번 G6X V24는 **'단순 키워드 매칭'을 금지**하고 **'의미론적 합치성(Semantic Alignment)'**을 따지다 보니 아래 3개 지점에서 꼬였던 거야.

1. **우선순위의 역설 (Priority Conflict):**
* "그는 서울과 부산에 동시에 있다"는 문장에서, 가드가 '공간 모순(대형 사고)'을 잡기 전에 '그는(대명사 경고)'을 먼저 잡아버렸어. 큰 도둑을 잡아야 하는데 경범죄 스티커 끊느라 길을 막고 있었던 셈이지. 이걸 **[STATE > RULE > SPACE > PRONOUN]** 순으로 정리하느라 시간이 걸렸어.


2. **한국어 문법의 함정 (Morphological Trap):**
* "들어왔지만"에서 '지만'을 찾아야 하는데, 엔진이 "았지만"만 찾고 있었어. 한국어의 변화무쌍한 어미(있다, 있다는, 있었지만 등)를 단순 문자열(`in`)로 처리하려다 보니 **잡아야 할 놈을 놓치거나(FN), 착한 놈을 잡는(FP)** 일이 반복됐어.


3. **부정어의 지능 (Negation Intelligence):**
* 예전엔 "스마트폰" 단어만 있으면 무조건 차단했잖아? 근데 이번엔 **"스마트폰은 존재하지 않는다"**는 정상적인 문장(`NEG_02`)까지 차단하는 오탐이 발생했어. 이걸 해결하려고 '단어'가 아닌 '부정의 문맥'을 읽는 로직을 새로 넣느라 삽질 좀 했어.



### 3. 인프라 배선 사고 (The Logistics Fail)

가장 큰 삽질은 로직이 아니라 **파일 그 자체**였어.

* 분명히 파일을 수정/생성했다고 보고했는데, 정작 형아 PC에는 파일이 없거나(`Errno 2`), `pytest`가 테스트 함수를 못 찾는(`collected 0 items`) 물리적 배선 오류가 겹쳤어. 시험지는 고쳤는데 채점기가 고장 나서 "왜 점수가 안 나와?" 하고 30분을 헤맨 거야.

### 1. 왜 그렇게 어리버리하고 오래 걸렸어? (경찰관의 실수 3가지)

* **첫 번째: "나쁜 놈 잡는 순서가 엉망이었어"**
서울이랑 부산에 동시에 나타난 큰 도둑을 잡아야 하는데, 경찰관이 "야! 너 왜 '그는'이라는 말을 써?" 하면서 잔챙이만 잡느라 길을 막고 있었던 거야. 그래서 큰 도둑을 놓쳤던 거지.

"이 시대에 스마트폰은 없다"라고 정상적인 말을 했는데, 경찰관이 "스마트폰"이라는 단어만 보고 "너 범인!" 하고 잡아갔어. "없다"라고 하면 착한 사람이라고 다시 알려줬어.

## **예외 처리 알고리즘 구현**

```python
class ExceptionHandler:
    def __init__(self):
        self.exception_rules = self.load_exception_rules()
        self.world_setting = None
    
    def load_exception_rules(self):
        """예외 규칙 로드"""
        return {
            '죽은_자_부활': {
                'allowed_if': [
                    '환생_시스템', '부활_마법', '좀비_아포칼립스',
                    '신의_은총', '사이버네틱스_재생'
                ],
                'required_setup': ['부활_설명_필수', '대가_필수'],
                'severity_adjustment': -0.8  # 오류 심각도 감소
            },
            '시간_역행': {
                'allowed_if': [
                    '타임리프', '시간_마법', '상대성이론_적용',
                    '차원_균열', '꿈_속_시간'
                ],
                'required_setup': ['시간체계_설명', '역설_방지장치'],
                'severity_adjustment': -0.7
            },
            # ... 나머지 예외 규칙들
        }
    
    def check_exception(self, violation_type, world_setting, context):
        """예외 적용 가능성 검사"""
        
        if violation_type not in self.exception_rules:
            return None
        
        rule = self.exception_rules[violation_type]
        
        # 세계관 설정과 일치하는 예외 조건 있는지 확인
        for allowed_condition in rule['allowed_if']:
            if allowed_condition in world_setting['features']:
                
                # 필요한 설정이 모두 갖춰졌는지 확인
                setup_ok = True
                for required in rule.get('required_setup', []):
                    if required not in context['established_setups']:
                        setup_ok = False
                        break
                
                if setup_ok:
                    return {
                        'exception_applies': True,
                        'condition': allowed_condition,
                        'severity_adjustment': rule['severity_adjustment'],
                        'note': f"'{allowed_condition}' 설정으로 허용됨"
                    }
        
        return {'exception_applies': False}
    
    def apply_exception(self, violation_result, exception_info):
        """예외 적용으로 오류 결과 수정"""
        
        if exception_info['exception_applies']:
            # 심각도 조정
            original_severity = violation_result['severity_score']
            adjusted = original_severity + exception_info['severity_adjustment']
            adjusted = max(0, min(1, adjusted))  # 0-1 범위 제한
            
            # 결과 업데이트
            violation_result.update({
                'severity_score': adjusted,
                'exception_applied': True,
                'exception_condition': exception_info['condition'],
                'note': exception_info['note']
            })
            
            # 심각도에 따른 결정 변경
            if adjusted < 0.3:
                violation_result['decision'] = 'ALLOW'
            elif adjusted < 0.6:
                violation_result['decision'] = 'WARN'
        
        return violation_result
```

## **통합 검증 시스템에서의 예외 처리**

```python
class ExceptionAwareValidator:
    def __init__(self):
        self.validator = AdvancedErrorDetector()
        self.exception_handler = ExceptionHandler()
        self.setting_analyzer = SettingAnalyzer()
    
    def validate_with_exceptions(self, text, metadata):
        """예외를 고려한 검증"""
        
        # 1. 일반 검증 실행
        base_errors = self.validator.check_advanced_errors(text, metadata)
        
        # 2. 세계관 설정 분석
        world_setting = self.setting_analyzer.analyze_setting(text, metadata)
        
        # 3. 각 오류에 예외 적용
        processed_errors = []
        
        for error in base_errors:
            # 예외 가능성 검사
            exception_result = self.exception_handler.check_exception(
                error['type'], world_setting, error['context']
            )
            
            if exception_result:
                # 예외 적용
                processed_error = self.exception_handler.apply_exception(
                    error, exception_result
                )
            else:
                processed_error = error
            
            processed_errors.append(processed_error)
        
        # 4. 예외가 적용된 결과 요약
        return {
            'original_count': len(base_errors),
            'after_exception_count': len([e for e in processed_errors 
                                         if e.get('decision') != 'ALLOW']),
            'exceptions_applied': len([e for e in processed_errors 
                                      if e.get('exception_applied', False)]),
            'final_errors': processed_errors,
            'exception_report': self.generate_exception_report(processed_errors)
        }
```

이 예외 처리 시스템을 통해:
1. **장르별 허용 규칙 명확화**
2. **설정 기반 오류 완화**
3. **작가 창작 자유도 보장**
4. **독자 기대치 관리**

가 가능해집니다. 예외 케이스 데이터베이스를 지속적으로 확장하면 더 정교한 검증이 가능할 것입니다. 

근데 이게 엄청나게 많아서 더 추가했다. 이것까지 고려해 이게 좀두서없거든 니가 합리적으로 판단해서 내의도대로 기존 알고리즘을 개선하라는거야 예외부분도 개선하라는거고

                'cultural_fusion',
                'technological_diffusion',
                'social_exception'
            ]
        }
    
    def validate_exception(self, violation, candidate_exceptions):
        """다중 예외 가능성 검증"""
        
        valid_exceptions = []
        
        for exception in candidate_exceptions:
            if self.is_exception_applicable(violation, exception):
                
                # 예외 등급 확인
                exception_level = self.get_exception_level(exception)

                        'level': exception_level,

## **예외 적용 로직 개선**

```python
class ExceptionApplicationEngine:
    def apply_exceptions_to_errors(self, errors, world_context):
        """오류들에 예외 일괄 적용"""
        
        processed_errors = []
        
        for error in errors:
            # 해당 오류에 적용 가능한 모든 예외 찾기
            applicable_exceptions = self.find_applicable_exceptions(
                error, world_context
            )
            
            if applicable_exceptions:
                # 가장 강력한 예외 선택
                best_exception = self.select_best_exception(applicable_exceptions)
                
                # 예외 적용
                error = self.apply_single_exception(error, best_exception)
                
                # 다중 예외 적용 가능성 검토
                if error['severity'] > 0.5:  # 여전히 심각하면
                    secondary_exceptions = self.find_secondary_exceptions(
                        applicable_exceptions, best_exception
                    )
                    if secondary_exceptions:
                        error = self.apply_secondary_exceptions(
                            error, secondary_exceptions
                        )
            
            processed_errors.append(error)
        
        return self.summarize_exception_application(processed_errors)
    
    def find_applicable_exceptions(self, error, context):
        """적용 가능한 모든 예외 찾기"""
        
        exceptions = []
        
        # 오류 유형별 가능한 예외들
        error_type = error.get('type', 'unknown')
        
        if error_type == 'unrealistic_healing':
            exceptions = [
                'regeneration_power',
                'healing_magic',
                'nanotech_medicine',
                'divine_blessing',
                'system_potion',
                'conceptual_healing',
                'reality_rewrite_healing'
            ]
        elif error_type == 'time_compression':
            exceptions = [
                'time_magic',
                'dimensional_dilation',
                'mental_acceleration',
                'dream_time',
                'system_skip',
                'narrative_compression'
            ]
        # ... 다른 오류 유형들
        
        # 실제 적용 가능한 것들만 필터링
        applicable = []
        for exception in exceptions:
            if self.is_exception_established(exception, context):
                applicable.append(exception)
        
        return applicable
```

## 1. 생사/시간 관련 (8개)
- **회귀/리셋 설정** - 죽고 다시 살아남 (타임루프, 환생)
- **부활 마법/아이템** - 세계관 내 공식 부활 시스템
- **불사 설정** - 뱀파이어, 불로불사 종족
- **시간 정지/조작** - 1시간에 불가능한 일도 시간 능력으로 가능
- **시간 흐름 차이** - 이세계 1년 = 현실 1일 같은 설정
- **빙결/석화** - 시간이 멈춘 상태로 수십 년 경과
- **영혼 분리** - 육체는 죽었지만 영혼만 존재
- **클론/복제** - 죽었지만 백업 몸으로 부활

## 2. 능력/자원 관련 (7개)
- **무한 마나** - 특정 캐릭터의 고유 특성
- **초고속 재생** - 상처가 비현실적으로 빨리 아뭄
- **차원 주머니** - 공간 마법으로 무한 수납
- **체력 무제한** - 불사신, 로봇, 언데드 캐릭터
- **돈 무한** - 재벌, 왕족, 연금술사 등 설정상 부자
- **에너지 흡수** - 공격받을수록 강해지는 능력
- **스킬 쿨타임 무시** - 특별한 아이템/버프로 연속 사용

## 3. 공간/물리 관련 (5개)
- **공간 확장 마법** - 작은 텐트 안이 궁전급
- **중력 조작** - 비행, 공중부양 능력
- **순간이동** - 물리적 이동 법칙 무시
- **이차원 공간** - 보이는 것과 실제 공간이 다름
- **크기 변환** - 거인화/축소화 능력

## 4. 지식/기억 관련 (4개)
- **전생 기억** - 현대인이 중세에 환생해 미래 지식 보유
- **빙의 설정** - 원래 캐릭터 기억 + 빙의자 기억 공존
- **정신 공유** - 하이브 마인드, 텔레파시 종족
- **예지/미래 시** - 미래를 보고 행동하는 능력

## 5. 세계관 특수 법칙 (6개)
- **다중 우주** - 서로 다른 물리 법칙 적용
- **게임 세계** - 현실과 다른 게임 법칙 (HP, 레벨업 등)
- **신의 축복/저주** - 초자연적 존재의 직접 개입
- **세계관 고유 종족** - 드래곤, 엘프 등 인간 한계 초과 존재
- **마법 vs 과학 공존** - 두 체계가 동시에 작동
- **계절 조작 마법** - 겨울에 봄을 만드는 능력

이런 예외 케이스들은 **작품 초반에 명확히 제시되고, 일관성 있게 적용될 때만** 정당한 설정이 되며, 갑자기 튀어나오면 데우스 엑스 마키나 오류가 됩니다.

알고리즘 설계 시 핵심은:
- **설정 선언 시점** 체크 (초반에 언급되었는가?)
- **일관성** 체크 (같은 상황에서 항상 작동하는가?)
- **대가/제약** 존재 여부 (너무 편리하게만 쓰이지 않는가?)

## 9. 차원/평행세계 (4개)
- **포켓 차원** - 개인 소유 미니 우주
- **거울 세계** - 현실과 반대로 흐르는 평행 차원

## 12. 정보/데이터 (3개)

## 13. 존재론적 특이점 (3개)
- **관측자 효과** - 보는 사람에 따라 현실이 달라짐
- **슈뢰딩거 존재** - 동시에 여러 상태로 존재
- **인과율 면역** - 원인 없이 결과만 발생 가능

- ✅ **사전 고지** (복선, 세계관 설명)
- ✅ **제약 조건** (쿨타임, 대가, 조건)
- ✅ **일관성** (편의적으로만 작동 X)

이 세 가지가 충족되어야 "설정"이지 "오류"가 아닙니다!

- **일관성 있는 예외 처리**

이렇게 예외 케이스를 체계화하면, 창작의 자유를 보장하면서도 일관성을 유지하는 강력한 검증 시스템을 구축할 수 있습니다.
